# 5层记忆系统修复 - 主控计划 (Master Plan)

## 🎯 最终目标
构建完整的记忆闭环：
```
对话输入 → L0捕获 → Daily Dream → L1生成 → Weekly Review → L2沉淀 
→ Monthly Review → L3框架 → 年度Review → L4核心 → 指导AI回答
```

## 📋 子任务分解 (4个并行进程)

### 进程A：L0基础设施修复 [P0]
**负责**：对话捕获机制
**输出**：`ai-memory-system/Memory/L0-state/daily-*.jsonl`
**验收标准**：
- [ ] 能捕获当前对话并写入L0
- [ ] 历史对话能回溯导入
- [ ] 数据格式符合规范

### 进程B：L1流向修复 [P0]
**负责**：Daily Dream → ai-memory-system 整合
**输出**：`ai-memory-system/Memory/L1-episodic/*.md`
**验收标准**：
- [ ] 每日23:00自动生成L1记忆
- [ ] 格式符合YAML Frontmatter规范
- [ ] 包含L2沉淀候选标记

### 进程C：L2/L3自动化 [P1]
**负责**：行为模式检测与沉淀
**输出**：`L2-procedural/*.md`, `L3-semantic/*.md`
**验收标准**：
- [ ] 检测重复模式(3次+)
- [ ] 自动标记候选
- [ ] 人工确认后自动更新

### 进程D：系统集成 [P0]
**负责**：AGENTS.md修改 + 记忆读取链
**输出**：修改后的AGENTS.md
**验收标准**：
- [ ] 每次对话前读取L4/L3/L2/L1
- [ ] 回答基于多层记忆综合
- [ ] 能引用具体记忆文件

## ⏱️ Heartbeat 监督机制

每30分钟检查一次：
```yaml
heartbeat_check:
  - 检查各子进程状态
  - 收集进度报告
  - 检测阻塞/失败
  - 必要时重启子进程
  - 汇报给用户
```
