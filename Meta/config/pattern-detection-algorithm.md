# 行为模式检测算法设计

## 1. 检测目标

| 层级 | 检测目标 | 触发条件 | Review周期 |
|------|---------|---------|-----------|
| **L2 行为层** | 重复性行为模式、习惯、偏好 | 同一模式出现 **3次+** | 每月 |
| **L3 认知层** | 思维模式、决策框架、心智模型 | 多L2指向同一框架 / 明确原则声明 | 每季度 |

---

## 2. L2 行为模式检测算法

### 2.1 输入数据源

```
L1-episodic/*.md  →  提取"观察到的行为模式"段落
                    ↓
               模式候选池
```

### 2.2 检测流程

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 模式提取                                             │
│  ─────────────────                                            │
│  从L1文件的"观察到的行为模式(L2候选)"段落提取条目                │
│                                                              │
│  格式示例：                                                    │
│  1. **系统化思维** — 要求清晰的层级结构和规则                   │
│     └─ 来源：2026-03-05-memory-system-init.md                 │
│                                                              │
│  2. **边界意识强** — 明确区分人工/AI权限边界                    │
│     └─ 来源：2026-03-06-multiagent-world-design.md            │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 相似度匹配                                           │
│  ─────────────────                                            │
│  对提取的模式进行语义相似度计算                                 │
│                                                              │
│  匹配维度：                                                    │
│  - 关键词重叠（Jaccard相似度 > 0.6）                           │
│  - 语义向量相似度（余弦相似度 > 0.85）                         │
│  - 行为描述的同义转换                                          │
│                                                              │
│  示例匹配：                                                    │
│  "喜欢清晰的层级结构" ≈ "要求明确的架构分层" ≈ "系统化思维"      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 频次统计与阈值判断                                     │
│  ─────────────────────────                                    │
│  对匹配到的模式组进行频次统计                                   │
│                                                              │
│  检测规则：                                                    │
│  IF count >= 3 AND time_span <= 30天 THEN                     │
│      → 标记为 L2-ready（候选沉淀）                            │
│                                                              │
│  例外情况（降低阈值）：                                         │
│  - 用户明确声明："我总是..." / "这是我的习惯..."               │
│  - 纠正信号：用户说"我都说了X遍了"                           │
│  → 直接标记为 L2-ready，count=1即可                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 置信度计算                                           │
│  ─────────────────                                            │
│  confidence = base_confidence × decay_factor × source_weight │
│                                                              │
│  base_confidence:                                             │
│    - 3次观察: 0.6 (medium)                                   │
│    - 5次观察: 0.8 (high)                                     │
│    - 用户明确声明: 0.9 (high)                                │
│                                                              │
│  decay_factor: 基于时间衰减                                    │
│    - 最近7天: 1.0                                            │
│    - 最近30天: 0.9                                           │
│    - 最近90天: 0.7                                           │
│                                                              │
│  source_weight:                                               │
│    - 纯观察: 0.8                                             │
│    - 用户声明: 1.0                                           │
│    - 混合来源: 0.9                                           │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 输出格式

```yaml
---
detection_id: L2-detected-20260402-001
detected_at: 2026-04-02T16:30:00Z
pattern_name: 系统化思维
pattern_category: thinking-style
count: 4
sources:
  - 2026-03-05-memory-system-init.md
  - 2026-03-10-architecture-design.md
  - 2026-03-15-api-design.md
  - 2026-03-20-data-model.md
time_span_days: 15
confidence: 0.85
confidence_level: high
status: pending_confirmation  # pending_confirmation / confirmed / rejected
target_file: Memory/L2-procedural/thinking-patterns.md
---

## 模式描述

**系统化思维** — 在多个场景中表现出对清晰结构和层级关系的偏好

### 观察实例
1. 设计AI记忆系统时要求5层架构
2. API设计时强调分层和接口边界
3. 数据模型设计时要求明确的实体关系
4. 工具配置时要求清晰的目录结构

### 行为特征
- 倾向于从整体到局部的思考方式
- 重视模块化和可扩展性
- 喜欢使用图表和层级结构表达想法

### 置信度依据
- 4次独立观察，时间跨度15天
- 跨多个不同场景（系统架构、API、数据、配置）
- 无反例观察
```

## 3. L3 认知模式检测算法

### 3.1 输入数据源

```
L2-procedural/*.md  →  提取稳定的行为模式
                      ↓
                 L2模式池
Intent/goals/*.md   →  目标与规划
                      ↓
                 目标意图池
                      ↓
               认知框架候选池
```

### 3.2 检测流程

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: L2聚合分析                                           │
│  ─────────────────                                            │
│  分析多个L2模式是否存在共同的底层框架                          │
│                                                              │
│  聚合类型：                                                    │
│  A. 多行为指向同一认知框架                                      │
│     例: "系统化思维" + "模块化偏好" + "接口边界意识"            │
│         → 指向认知框架: "架构优先的设计哲学"                   │
│                                                              │
│  B. 行为-目标一致性                                            │
│     例: L2"晨间规划习惯" + Intent"提升工作效率目标"            │
│         → 指向决策框架: "时间盒管理方法"                       │
│                                                              │
│  C. 明确的原则声明                                             │
│     用户说: "我的原则是..." / "我倾向于..."                   │
│         → 直接标记为L3候选                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 框架抽象                                             │
│  ─────────────────                                            │
│  从聚合的模式中抽象出更高层的认知框架                          │
│                                                              │
│  抽象层次：                                                    │
│  - Level 1: 具体行为 (L2)                                     │
│  - Level 2: 习惯组合                                          │
│  - Level 3: 认知框架 ← 目标层级                                │
│                                                              │
│  抽象示例：                                                    │
│  "Tavily搜索偏好" + "工具配置要求" + "脚本标准化"              │
│      ↓                                                       │
│  工具使用中的可复现性追求                                       │
│      ↓                                                       │
│  【认知框架】工程化思维 — 追求流程的可重复性和可自动化           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 稳定性检验                                           │
│  ─────────────────                                            │
│  检验认知框架的稳定性                                          │
│                                                              │
│  稳定性指标：                                                  │
│  - 跨时间稳定性: 90天内持续表现                                │
│  - 跨场景一致性: 在3+不同场景中出现                            │
│  - 反例缺失: 无显著矛盾行为                                    │
│                                                              │
│  检测规则：                                                    │
│  IF stability_score >= 0.8 THEN                              │
│      → 标记为 L3-ready（候选沉淀）                            │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 L3 沉淀触发条件

| 触发类型 | 条件 | 示例 |
|---------|------|------|
| **聚合触发** | 3+ L2模式指向同一框架 | 多个工作习惯 → 效率优先原则 |
| **声明触发** | 用户明确表达思维/决策模式 | "我倾向于先分析再行动" |
| **稳定触发** | L2模式持续3个月+无变化 | 每月review确认的固定习惯 |
| **纠正触发** | 用户纠正AI对其思维的理解 | "你理解错了，我其实是..." |

## 4. 实现脚本规范

### 4.1 检测脚本接口

```bash
# 执行L2模式检测
node scripts/detect-l2-patterns.mjs \
    --input-dir Memory/L1-episodic/ \
    --output-dir Meta/insights/ \
    --threshold 3 \
    --time-window 30

# 输出：
# - Meta/insights/l2-candidates-YYYY-MM-DD.json
# - 控制台报告
```

```bash
# 执行L3模式检测
node scripts/detect-l3-patterns.mjs \
    --l2-dir Memory/L2-procedural/ \
    --intent-dir Intent/ \
    --output-dir Meta/insights/ \
    --stability-threshold 0.8

# 输出：
# - Meta/insights/l3-candidates-YYYY-MM-DD.json
# - 控制台报告
```

### 4.2 配置文件

```yaml
# Meta/config/pattern-detection.yaml
detection:
  l2:
    threshold: 3                    # 最少触发次数
    time_window_days: 30            # 时间窗口
    similarity_threshold: 0.85      # 语义相似度阈值
    keyword_overlap: 0.6            # 关键词重叠阈值
    
  l3:
    stability_window_days: 90       # 稳定性检验窗口
    min_l2_patterns: 3              # 聚合所需最少L2模式
    cross_scene_min: 3              # 跨场景最小数
    
  confidence:
    observation_weight: 0.8
    declaration_weight: 1.0
    decay_7d: 1.0
    decay_30d: 0.9
    decay_90d: 0.7
```

## 5. 算法边界与限制

### 5.1 已知限制

1. **语义理解局限**
   - 无法100%准确识别同义表达
   - 依赖关键词和向量相似度，可能有误判

2. **时间窗口约束**
   - 30天/90天窗口可能错过慢速变化的模式
   - 季节性行为可能被错误归类

3. **置信度校准**
   - 初始阈值基于启发式，需要实际运行调优

### 5.2 安全边界

- **不自动写入L2/L3**：所有检测仅为候选，需人工确认
- **L4永不自动检测**：核心价值观/身份必须由用户直接设定
- **提供反例机制**：用户可标记"这不是我"，加入黑名单
