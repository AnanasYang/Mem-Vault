# 系统化思维偏好

在多个设计和决策场景中表现出对清晰层级结构、模块化设计和明确边界的偏好。

---

## 模式定义

### 核心特征
- **层级化思考**: 倾向于从整体架构到具体实现的自顶向下思考
- **模块化设计**: 重视组件的独立性、可复用性和可替换性
- **边界意识**: 强调接口、职责和边界的清晰划分
- **结构化表达**: 喜欢使用图表、列表和层级结构来组织和呈现信息

### 触发场景
- 系统架构设计
- API/接口设计
- 数据模型设计
- 工具/脚本配置
- 流程规范制定

### 典型表现
1. 设计时先定义整体架构，再细化各层
2. 要求每个模块有明确的输入/输出边界
3. 配置文件要求清晰的目录结构和命名规范
4. 文档编写时使用层级标题和结构化列表

## AI应用指南

### 当检测到该模式时
1. 优先提供结构化的方案（列表、层级、流程图）
2. 强调架构设计和边界划分
3. 询问对整体架构的看法，再进入细节

### 建议的响应方式
- "从架构层面来看，我们可以分为以下几个层次..."
- "这个模块的职责边界可以这样划分..."
- 使用Markdown表格、列表呈现选项对比

### 需要避免的行为
- 直接提供代码实现而不先说明架构
- 模糊的、非结构化的表达
- 忽略模块间的边界和接口设计
